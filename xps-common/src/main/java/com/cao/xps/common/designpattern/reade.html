设计模式 1、单例模式：一个对象只实例化一次 饿汉模式： 1、将构造方法私有化 2、通过静态方法实例化对象 优：现拿现用 缺：不管有没有用到都会创建实例，耗费资源 懒汉模式： 1、将构造方法私有化 2、静态实例一个空对象，使用是给空对象赋值 优：节约资源 缺：有线程安全问题，加锁影响速度 最佳：1、私有化构造方法；2、创建匿名内部类，在匿名内部类中静态加载构造方法 2、工厂模式 简单工厂：定义一个专注于创建对象的类 普通工厂：每个类创建一个对应的工厂，用工厂来创建类 抽象工厂：创建一个抽象类，类中多个获取对象的方法，返回值都为接口或父类，工厂类继承该抽象类，实现工厂类中不同对象的组合使用 3、策略模式 例：通过对象的某个值进行排序 思路：1、定义两个接口，策略接口定义策略方法，参数为两个任意对象；方法接口定义一个方法，参数为任意对象数组和策略接口 2、实现策略接口，通过对比两个参数的值，返回三个状态-1、0、1 3、实现方法接口，通过循环对象数组，在循环中通过策略接口返回值状态，调整数组顺序； 4、代理 静态代理 例：日志代理和时间代理 1、定义一个接口，接口中实现需要代理的方法 2、被代理对象和代理对象都实现接口 3、代理对象中将接口作为参数，实现代理方法后在代理方法中调用接口中的方法，在调用前后实现代理 动态代理 JDK动态代理： 1、定义一个接口 2、被代理对象实现接口 3、通过Proxy类中getProxyInstance方法，实现动态代理,其中有invok方法 CJLB动态代理：通过Enhancer对象实现动态代理 区别：JDK动态代理需要实现接口，CJLB不需要 5、观察者模式 将事件封装为一个对象，包含事件的状态 6、迭代模式 通过集合类共同继承同一接口，接口中定义获取下个对象的方法和获取总数的方法，使得循环不再关注对象类型 7、树结构模式 1、创建一个抽象类 2、创建两个实体类继承同一抽象类，一个只包含对象属性，另一个将抽象类作为实体类参数，内部存放list抽象对象并提供add方法
